# Reproducible build instructions

These steps document a minimal, reproducible build for creating Python
sdist/wheel artifacts locally. CI/release workflows already build and
publish artifacts with attestations and an SBOM.

1. Create and activate a virtual environment:

```bash
python -m venv .venv
.venv/Scripts/activate      # Windows
# source .venv/bin/activate # Linux/macOS
```

2. Install build tools (use the pinned release-tools if available):

```bash
python -m pip install --upgrade pip build
if [ -f requirements/release-tools.txt ]; then python -m pip install -r requirements/release-tools.txt; fi
```

3. Build sdist and wheel:

```bash
python -m build
```

4. Verify produced artifacts and SBOM (CI attaches SBOM and attestations):

```bash
ls dist/
cat sbom.cdx.json  # if generated by release workflow
```

5. For reproducible builds across environments, use the same Python
   micro-version and the pinned `requirements/*` files used in CI.
